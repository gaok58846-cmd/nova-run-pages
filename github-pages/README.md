# NOVA RUN

无需安装、无需后端，可直接通过 GitHub Pages 运行的霓虹跑酷游戏。

## 当前功能

- 中文、English、Русский 三种语言
- 简单、普通、困难三档难度及独立最高分
- 六个可选场景
- 二段跳、滑铲、能量冲刺、完美闪避
- 护盾、磁铁、减速与超载状态
- 桌面键盘、手机触控、横竖屏与全屏降级
- 暂停和声音开关
- 可复现挑战种子：`?seed=challenge-42`
- 按物理能力计算的安全障碍间距与四阶段难度曲线
- 六个场景各自独立的障碍池和安全组合模板
- 六张地图拥有独立玩法：
  - 广州：低空无人机、移动广告牌和桥梁横杆，节奏最适合入门
  - 上海：高速列车、移动闸门以及跳跃接滑铲组合
  - 深圳：激光、机器人、扫描器；冲刺可关闭电子机关
  - 雪山：受控冰面前滑、雪球、冰柱和断裂冰面
  - 火山：带提前预警的落石、火焰柱和熔岩区，轻量热浪与火星
  - 九寨沟：倒木、水面跳台、瀑布水流和断裂木桥，轻量水面倒影
- 地图设置使用七张独立的原创场景预览图；预览统一只展示地标、路线与机关，不放人物，避免不同生成图出现角色身份漂移。
- 地图选择分成“无尽跑酷”和“闯关模式”两个分页。六张自动奔跑地图位于无尽模式，`晨雾折光峰` 作为独立手工关卡只出现在闯关模式。

## 障碍调试

- 正常地址不带 `seed` 参数，每局使用全新随机序列。
- 在地址后加入 `?seed=任意文字` 可固定障碍序列；“再来一局”会从同一个序列起点重新生成。
- 种子模式游玩时，左侧 HUD 会显示当前种子、生成阶段和方案编号。
- 生成阶段依次为适应期、早期、进阶和后期；适应期及早期只生成单一障碍，随后才逐渐加入场景专属组合。

## 操作

- `空格` / `W` / `↑`：跳跃与二段跳；提前松开可缩短跳跃
- `S` / `↓`：滑铲
- `Shift` / `K` / `→`：消耗能量冲刺
- `P` / `Esc`：暂停或继续
- `M`：开关声音

## 项目结构

- `index.html`：语义化页面与游戏控件
- `styles.css`：主题、响应式布局与触控安全区
- `config.js`：难度、物理、场景和公平性参数
- `i18n.js`：中英俄文案
- `game.js`：游戏状态、输入、物理、障碍和渲染
- `tests/`：核心参数、生成确定性、理论可通过性、兼容性和全场景十分钟模拟测试

项目不使用广告、追踪脚本、大型框架或商业游戏素材。

## 第四阶段：视觉与反馈

- 主角采用原创霓虹跑者造型，并为跑步、起跳、二段跳、滑铲、冲刺和失败分别绘制动作姿态；步频、跨幅、摆臂、身体前倾和滑铲伸展会随当前速度及难度同步变化。
- 障碍统一使用动作提示：橙色 `↑` 表示跳跃、蓝色 `↓` 表示滑铲、绿色 `⚡` 表示冲刺；支持两种解法的机关会同时显示两个符号。
- 高速时加入轻量速度线；完美闪避使用短促扩散环和双音反馈；超载统一使用绿色边框、残影、速度线和双段提示音。
- HUD 改为顶部两张紧凑信息卡，在小屏横竖屏下避开角色、跑道、方向按钮和触控操作区。
- 主菜单改为独立 HTML 路线控制台，场景、难度、玩法摘要、开始、成长和设置都有固定响应式网格；不再叠加画布假按钮和菜单浮动按钮。暂停、结算与设置沿用同一套暖色编辑式几何界面，结算页会突出新纪录。
- 设置中新增“减少动态效果”，会关闭屏幕震动、闪屏、速度线和大部分粒子，并降低装饰动画；设置会保存在本机。

## 第五阶段：本地内容

- 首次奔跑提供五步可跳过教程：跳跃、滑铲、二段跳、冲刺破障和收集能量；完成状态保存在本机。
- 本地成长记录包含总距离、局数、最高连击、总收集量、六项成就、七张逐步解锁场景和五种纯外观配色/拖尾。所有解锁都不改变速度、跳跃或能量数值。
- 每天按设备本地日期生成距离、完美闪避和冲刺破障三项任务，不连接服务器。
- HTML 结算面板显示分数、距离、最高分、最高连击、收集数量、场景、难度和新纪录，并提供重试、菜单、换场景和分享。
- 分享优先调用 Web Share API；`file://` 或不支持系统分享的浏览器会打开可见文本框，可一键复制或手动长按复制。链接只包含 `scene`、`difficulty` 和随机 `seed`。
- 程序化 Web Audio 提供可选的 96 BPM 原创氛围配乐：低通和弦铺底、轻量拨弦、柔和底鼓与稀疏高频节奏代替重复提示音；背景音乐默认关闭，且只有用户开始游戏后才创建/恢复音频上下文。
- 设置保存音乐/音效、总音量、分项音量、震动、屏幕震动强度、减少动态效果、高对比度、触控透明度和教程开关；清档操作需要二次确认。
- 显示模式可在暗夜与日间主题之间切换并保存；两套主题分别调整画布、按钮、菜单和设置面板的文字对比度。
- 顶部品牌使用原创六边形轨道、冲刺 `N`、星点和倾斜字标组合；主菜单显示完整徽标，游玩时缩为安全区紧凑版，`MADE BY KAI` 使用独立署名徽章。

第五阶段继续保持零后端、零广告、零追踪、零付费内容。新增模块：

- `storage.js`：存档、统计、解锁、成就和每日任务。
- `audio.js`：用户交互后启动的程序化音乐与音效。
- `ui.js`：教程、成长面板、设置、结算、分享和清档确认。
- `player.js`：角色与拖尾渲染。

## 晨雾折光峰（壁纸改编平台关）

场景选择中新增 `晨雾折光峰`。该地图从暖米色雾空、白色日轮、橙色受光山脊、深灰几何山体和稀疏荒原元素重新设计而来，没有直接把壁纸当作静态背景。

该地图现已从无尽跑酷列表中独立出来：地图选择器切换到“闯关模式”后才显示它；主菜单会改为“关卡闯关”和“开始闯关”，并隐藏无效的无尽模式难度选项。

关卡是一条约 7200px 的手工路线：

1. 宽平台教学区：先熟悉左右移动和宽容跳跃。
2. 单机制区：依次介绍移动平台、踩落平台、弹跳平台与传送门。
3. 环境区：风区提供轻推和上升气流，重力区降低重力但不反转操作。
4. 机关区：收集三枚山光碎片打开第一扇门，再踩下 8 秒限时开关通过第二扇门。
5. 组合区：用检查点保护移动平台、踩落平台和风区组合；高空隐藏山脊提供额外收集物和返回传送门。
6. 峰顶终点：抵达发光圆环完成关卡。

平台主路线宽度不低于 150px，基础间距不超过 150px，并在长距离之间布置落脚点。关卡开始后人物与肢体动画都保持静止，只有按 `A/D`、方向键或触屏左右按钮才会移动并进入跑步动作；空格跳跃、`Shift` 冲刺。

主菜单封面使用项目内原创生成插画：暖米色天空、陶土橙、深炭建筑、远景城市与折光山峰。桌面版加载约 100KB 的横版 WebP，手机竖屏加载约 36KB 的专用裁切版本。

所有场景使用同一套立即死亡规则：DeathZone 位于基础地面下方 40px。新平台关掉入 DeathZone 或碰到山石尖刺时会立即在最近检查点复活；六张跑酷地图每 500 米记录自动安全检查点，掉出路线后立即清除近处危险并复活，不再等待角色长时间下坠，也不会让背景突然倒退。
